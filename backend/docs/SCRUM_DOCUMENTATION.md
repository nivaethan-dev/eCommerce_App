# **3. Sprint Plan**

## Sprint 1

**Sprint Goal:** Build foundational user authentication, secure setup, and product browsing.

**Sprint Duration:** Weeks 1–2

**Planned Daily Activities:**
- Day 1-2: Setup project structure, MongoDB connection, environment configuration, and CORS setup
- Day 3-4: Implement Customer, Admin, and Product schemas with proper validation and indexes
- Day 5-6: Build user registration with input validation (email, phone, password strength)
- Day 7-8: Implement JWT login with httpOnly cookies, refresh tokens, and token rotation
- Day 9-10: Add rate limiting for login and registration, create unified login endpoint
- Day 11-12: Build product listing API with search and query helper utilities
- Day 13-14: Create frontend pages (Home, Login, Signup), Header, Footer, Layout components

## Sprint 2

**Sprint Goal:** Enable cart, orders, and admin product management.

**Sprint Duration:** Weeks 3–4

**Planned Daily Activities:**
- Day 1-2: Implement persistent cart with add/update/remove operations
- Day 3-4: Add stock validation and MongoDB transactions for cart
- Day 5-6: Create cart summary with tax calculations (8% rate in LKR)
- Day 7-8: Build order placement with atomic stock decrement and price snapshots
- Day 9-10: Implement admin product creation with image upload and processing
- Day 11-12: Create admin order and customer management with RBAC
- Day 13-14: Add database seeders, custom error classes, and message constants

## Sprint 3

**Sprint Goal:** Complete search, notifications, UI polish, and security hardening.

**Sprint Duration:** Weeks 5–6

**Planned Daily Activities:**
- Day 1-2: Implement product search across title, description, category fields
- Day 3-4: Add pagination for product and order listings
- Day 5-6: Enforce RBAC on all admin routes
- Day 7-8: Create notification UI placeholder and order confirmation messages
- Day 9-10: Finalize responsive UI with mobile/tablet breakpoints
- Day 11-12: Add dark mode support and CSS animations
- Day 13-14: Write unit/integration tests and perform final bug fixes

# **4. Increment Summary**

## Sprint 1 Increment

**Frontend:** Home page with hero section and category showcase, login page with form validation, signup page with form validation, header component with navigation and search bar, footer component, layout component, API utility functions, responsive CSS foundation.

**Backend:** MongoDB schemas for Customer (with embedded cart), Admin, and Product, user registration with email/phone/password validation, unified JWT login for customer and admin, logout functionality, httpOnly cookie storage, refresh token with rotation, rate limiting for login and registration, auth middleware, product listing API with search, query helper utility, token and security utilities, CORS and environment configuration.

## Sprint 2 Increment

**Frontend:** Cart icon with item count badge, profile dropdown with logout, notification bell placeholder, auth state management with custom events.

**Backend:** Persistent shopping cart with full CRUD operations, stock validation preventing overselling, MongoDB transactions for cart and orders, cart calculations with subtotal/tax/total, order placement with atomic stock decrement, order schema with price snapshots, customer order history, admin product creation with image upload, image processing with Sharp (800x600 resize, JPEG conversion), image validation using file-type, admin order viewing with pagination and status filter, admin order status update, admin customer viewing, RBAC middleware, custom error classes, centralized message constants, database seeding scripts, static file serving, customer/order/product service layers.

## Sprint 3 Increment

**Frontend:** Responsive UI for mobile and tablet, dark mode support via CSS media query, animations and hover effects, accessible components with ARIA labels, Vite proxy configuration, production build setup.

**Backend:** Pagination for products and orders, RBAC enforcement on admin routes.
